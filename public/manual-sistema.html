1. Registro de Pacientes

1.1 Registro Único

Cada paciente se identifica de forma única por su Tipo de Documento + Número de Documento (ej: CC 1098765432). El sistema impide la creación de pacientes duplicados.

Si se intenta registrar un paciente con un documento ya existente, el sistema muestra un mensaje de error indicando el nombre del paciente registrado previamente.

1.3 Protección de Datos del Paciente

⛔ PROHIBIDO: Los datos del paciente no se pueden eliminar ni editar directamente. El tipo y número de documento son inmodificables. Las correcciones se realizan exclusivamente mediante Notas Aclaratorias (ver sección 3).

1.4 Ficha del Paciente

Al consultar un paciente, el sistema muestra:

  • Todos sus datos personales y de salud
  • Cantidad total de Historias Clínicas registradas
  • Cantidad total de Órdenes de Servicio asociadas
  • Indicador 🛡️ Registro protegido

2. Historias Clínicas APH — Inmutabilidad

2.1 Marco Legal

Resolución 1995 de 1999 — Ministerio de Salud de Colombia:
La historia clínica es un documento privado, obligatorio e inmodificable. Todo profesional que intervenga en la atención del paciente tiene la obligación de registrarla y conservarla. No se permite la alteración, destrucción ni eliminación de los registros.

2.2 Funcionamiento en el Sistema

Cada atención prehospitalaria genera una nueva historia clínica vinculada al paciente. Las historias se acumulan — un mismo paciente puede tener múltiples registros a lo largo del tiempo.

⛔ PROHIBIDO:
• Editar una historia clínica existente
• Eliminar una historia clínica
• Modificar cualquier campo después de guardar
El sistema no tiene botones de editar ni eliminar para historias clínicas.

3. Notas Aclaratorias — Correcciones Legales

3.1 ¿Qué son?

Las notas aclaratorias son el único mecanismo legal para corregir errores en una historia clínica ya registrada, conforme a la Resolución 1995/99.

No se modifica el registro original. En su lugar, se crea un documento adicional que aclara el error.

3.2 Contenido de una Nota Aclaratoria

CampoDescripción
Historia ClínicaID de la HC a la que se refiere la corrección
MotivoRazón de la corrección
Campo AfectadoQué campo contiene el error
Valor AnteriorEl dato erróneo original
Valor CorrectoEl dato correcto
Médico ResponsableNombre del profesional que autoriza la corrección
Fecha y HoraRegistrada automáticamente por el sistema
⚠️ Importante: Solo profesionales con rol de Médico pueden crear notas aclaratorias. Las notas también son inmutables — una vez creadas, no se pueden editar ni eliminar.

3.3 Ejemplo Práctico

Situación: Se registró al paciente con Triage "III" pero debía ser "II".

Solución: Se crea una Nota Aclaratoria:
• Campo: Triage
• Valor anterior: III — Urgencia
• Valor correcto: II — Emergencia
• Motivo: Error de clasificación al momento del registro
• Médico: Dr. Carlos Ramírez

4. Protección de Datos y Auditoría

4.1 Registro de Auditoría

Toda modificación realizada en los datos de un paciente genera un log de auditoría inmutable que registra:

  • Qué campo fue modificado
  • Cuál era el valor anterior
  • Cuál es el nuevo valor
  • Quién realizó el cambio
  • Fecha y hora exacta

4.2 Ante una Auditoría Gubernamental

✅ El sistema permite:
1. Buscar al paciente por su número de documento
2. Ver todos sus datos desde el primer registro
3. Consultar todas las historias clínicas de todos los atendimientos (acumulativas)
4. Ver las notas aclaratorias vinculadas a cada historia
5. Revisar el log de auditoría con todas las modificaciones

Nada se pierde. Todo queda registrado permanentemente.

5. Roles y Control de Acceso

5.1 Marco Legal — Ley 1581 de 2012

Ley 1581 de 2012 — Protección de Datos Personales:
Los datos clínicos son considerados datos sensibles. Su tratamiento requiere autorización expresa del titular y solo puede ser realizado por personal autorizado. El responsable del tratamiento debe implementar medidas de seguridad para evitar su acceso no autorizado, adulteración, pérdida, consulta o uso no autorizado.

5.2 Roles del Sistema

El sistema implementa 3 roles con permisos diferenciados para proteger los datos clínicos sensibles:

RolDescripción¿Quién lo usa?
🛡️ Administrador Acceso total al sistema + gestión de usuarios y roles Director de operaciones, gerente
🩺 Médico / TPH Acceso a pacientes completos, historias clínicas, signos vitales, notas aclaratorias Médicos, tecnólogos en atención prehospitalaria, enfermeros
🚑 Conductor Solo órdenes de servicio y ambulancias. No tiene acceso a datos clínicos Conductores de ambulancia

5.4 ¿Cómo funciona?

  • Cada usuario registrado tiene un rol asignado almacenado en la base de datos
  • El menú lateral (sidebar) muestra solo las opciones permitidas según el rol
  • Las reglas de seguridad de Firestore impiden el acceso directo a datos protegidos, incluso si alguien intenta saltar la interfaz
  • Solo el Administrador puede cambiar el rol de otros usuarios
⛔ IMPORTANTE: Un conductor que intente acceder a una historia clínica (por URL directa o manipulación) será bloqueado por las reglas de seguridad tanto a nivel de interfaz como a nivel de base de datos.

6. Funcionamiento Offline

El sistema está diseñado para operar en condiciones de conectividad limitada, ideal para ambulancias en zonas rurales o con señal inestable.

5.1 ¿Cómo funciona?

  • Los datos se almacenan en caché local del dispositivo
  • Si no hay internet, el usuario puede seguir registrando pacientes e historias clínicas
  • Cuando se recupera la conexión, todo se sincroniza automáticamente con el servidor
  • El sistema muestra un indicador 📴 Sin conexión cuando está offline

7. Seguridad del Sistema

6.1 Autenticación

  • Acceso solo para usuarios autenticados
  • Inicio de sesión con correo/contraseña o Google
  • Sesión protegida con tokens JWT

6.2 Protección a Nivel de Base de Datos

Las reglas de seguridad se aplican directamente en Firebase/Firestore, lo que significa que incluso si alguien intentara acceder al banco de datos directamente (sin la aplicación), las restricciones de inmutabilidad y eliminación se mantienen.

9. Preguntas Frecuentes

¿Se puede eliminar un paciente?

No. Los datos del paciente son permanentes. Si ya no es atendido, permanece en el sistema con su historial completo.

¿Se puede editar una historia clínica?

No. Una vez registrada, la HC es inmutable. Las correcciones se hacen con Notas Aclaratorias.

¿Qué pasa si el paciente ya fue atendido antes?

Se crea una nueva historia clínica vinculada al mismo paciente. El sistema acumula todas las HC, sin sobrescribir las anteriores.

¿Qué pasa si no hay internet?

El sistema funciona offline y sincroniza cuando se recupera la conexión.

¿Cómo se prepara para una auditoría?

No se necesita preparación especial. Todo está registrado de forma permanente e inmutable. Solo busque al paciente por su documento y toda la información estará disponible.
